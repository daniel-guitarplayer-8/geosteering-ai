## Auditoria — 3 Patologias Identificadas

### Patologia 1 — Cold-Start 35-119× mais lento

| Campo | Detalhe |
|:--|:--|
| **Descrição** | Run 1 (vmap) é 35–119× mais lento que Runs 2-5 em cenários B, C, E, H |
| **Causa-raiz** | `_BUCKET_JIT_CACHE` em `forward_pure.py:573` keyed `(camad_t, camad_r, n, npt)` — 50 modelos aleatórios geram ~50 entradas distintas; warmup com `models[0]` cobre apenas O(1) bucket |
| **Referência de código** | `geosteering_ai/simulation/_jax/forward_pure.py:573` |
| **Mitigação** | `vmap_real` usa `_UNIFIED_JIT_CACHE` keyed `(n, npt)` — invariante a modelos, sem cold-start |

### Patologia 2 — JAX GPU mais lento que Numba CPU

| Campo | Detalhe |
|:--|:--|
| **Descrição** | JAX GPU hot-cache 2-5× mais lento que Numba CPU 8C/16T |
| **Causa-raiz 1** | Loop Python serial sobre modelos (`multi_forward.py:400-422`) — GIL preso, CPU-bound |
| **Causa-raiz 2** | `build_static_context()` recriado por modelo: 5-20ms overhead Python (Hankel I/O + 8× `jnp.asarray` + `find_layers_tr` Numba FFI) |
| **Causa-raiz 3** | `np.asarray(H_jax)` força GPU→CPU sync por modelo — mata async pipelining |
| **Causa-raiz 4** | Ausência de `vmap` sobre eixo de modelos — GPU subutilizada (compute <1ms vs overhead 5-20ms) |
| **Referência de código** | `geosteering_ai/simulation/_jax/multi_forward.py:400-422` |

### Patologia 3 — Hardware Mismatch no Baseline

| Campo | Detalhe |
|:--|:--|
| **Descrição** | Baseline Numba medido em **Intel i9-9980HK 8C/16T (Mac Intel)** — `.claude/perf_baseline.json:15`, `docs/reports/v2.36_2026-05-15.md:8`. T4 tem n1-standard-4 4vCPU sem HT |
| **Impacto** | Gate ≥1.5× Numba não é hardware-aware; comparativo é JAX-GPU-T4 vs Numba-CPU-i9-9980HK |
| **Remediação** | Re-medir Numba localmente no T4 via `geosteering-cli benchmark` antes de revalidar gate (Sprint A1.6) |
