## Contexto

Sprint A1 (`A-jax-gpu-validate`) fechou **DONE-PARTIAL**: paridade Fortran
<1e-12 confirmada em Colab T4, mas gate de performance falhou (A: 0.38×,
B: 0.37×, E: 0.61× Numba). Causa-raiz arquitetural: assinatura de
`simulate_multi_jax` aceitava apenas **um modelo por chamada**, forçando
o benchmark notebook a iterar `for model in models:` em Python. Para
n_pos ≤ 600, o compute JAX GPU é <1 ms, mas overhead Python por modelo
é 5-20 ms — a GPU ficava ociosa >90% do tempo.

Esta sprint resolve a causa-raiz introduzindo
`simulate_multi_jax_batched()` — API batched que aplica `jax.vmap` sobre
o eixo `n_models`.

## Design Arquitetural

### Composição vmap Dupla

```
┌─────────────────────────────────────────────────────────────────┐
│  simulate_multi_jax_batched(rho_h_batch, ...)                  │
│                                                                 │
│  ┌── Pré-computa (1× por batch, NumPy) ──────────────────┐    │
│  │  Hankel filter weights, krJ0J1, wJ0, wJ1, npt        │    │
│  │  _sanitize_profile_batch(n, esp_batch)               │    │
│  │    → h_arr_batch (n_models, n)                        │    │
│  │    → prof_arr_batch (n_models, n+1)                   │    │
│  └────────────────────────────────────────────────────────┘    │
│                                                                 │
│  jax.vmap(_one_model, in_axes=(0, 0, 0, 0))   ← VMAP EXTERNO  │
│  └── _one_model(rho_h, rho_v, h_arr, prof_arr)                │
│      │                                                          │
│      jax.vmap(_one_config, in_axes=(0, 0))  ← VMAP INTERNO    │
│      └── _one_config(L, theta_deg)                            │
│          │                                                      │
│          find_layers_tr_jax_vmap(Tz, Rz, prof_arr, n)         │
│          jitted = _get_unified_jit(n, npt)  ← Sprint 10 cache │
│          jitted(rho_h, rho_v, ..., camad_t, camad_r, ...)     │
│                                                                 │
│  H_tensor_jax.block_until_ready()  ← ÚNICO sync (não n×)      │
│  np.asarray(H_tensor_jax)          ← ÚNICA conversão         │
└─────────────────────────────────────────────────────────────────┘
```

### Por que isto resolve as 3 causas-raiz da Sprint A1

| Causa-raiz | Antes (loop Python) | Depois (batched) |
|:--|:--|:--|
| Bucket cache explosion | 50 modelos × ~50 compilações XLA (key inclui `ct, cr`) | 1 compilação XLA via `_UNIFIED_JIT_CACHE` (key `(n, npt)`) |
| Overhead Python serial | `build_static_context()` recriado por modelo (5-20 ms × 50 = 0.25-1 s) | Static context compartilhado; sanitize <50 ms total |
| GPU→CPU sync por modelo | 50 `np.asarray()` serializados (mata async pipelining) | 1 `block_until_ready()` + 1 `np.asarray()` ao final |

## API Pública

```python
from geosteering_ai.simulation import simulate_multi_jax_batched

# Input: arrays 2D (n_models, n) — todos modelos compartilham n
result = simulate_multi_jax_batched(
    rho_h_batch=rho_h_arr,    # shape (n_models, n) Ω·m
    rho_v_batch=rho_v_arr,    # shape (n_models, n) Ω·m
    esp_batch=esp_arr,        # shape (n_models, n-2) m
    positions_z=positions,    # shape (n_pos,) — compartilhado
    frequencies_hz=[20e3, 80e3],
    tr_spacings_m=[0.5, 1.0],
    dip_degs=[0.0, 30.0, 60.0],
)
# Output: MultiSimulationResultBatchedJAX
# H_tensor shape: (n_models, nTR, nAngles, n_pos, nf, 9) complex128

# Extrair modelo individual:
single = result.get_model(i_model=2)  # → MultiSimulationResultJAX
```

### Restrição arquitetural

**Todos os modelos do batch DEVEM compartilhar o mesmo `n` (n_camadas).**
Para `n` heterogêneo, agrupar por `n` e chamar separadamente. Decisão
arquitetural: `prof_arr` tem shape `(n+1,)`, `np.stack` requer `n`
homogêneo para vmap compor corretamente.
