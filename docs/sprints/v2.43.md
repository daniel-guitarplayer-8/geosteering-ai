## Métricas Reportadas pelo Notebook (esperadas em T4)

Cada cenário reporta:
- `run_1_cold_mod_h`: throughput Run 1 (deve ser ≈ hot se warmup efetivo)
- `hot_throughputs_mod_h`: 4 throughputs (Runs 2-5)
- `median_hot_mod_h`: `statistics.median` — métrica oficial
- `ratio_cold_to_hot`: cold/hot — esperado ≈ 1.0 (diagnóstico)
- `latency_ms_per_model_hot`: `3.6e6 / median_hot` ms (CRIT-1 fix)
- `numba_t4_local_mod_h`: subprocess `geosteering-cli benchmark` (M3 fix)
- `numba_baseline_historical_mod_h`: i9-9980HK (informativo)

**Gate**: `median_hot / numba_t4_local ≥ 1.5` em A, B, E. H excluído (M1).

| D1 | Comparar vmap legacy vs batched no notebook? | **Não** — regression test em pytest T15 cobre paridade; notebook foca em batched only |
