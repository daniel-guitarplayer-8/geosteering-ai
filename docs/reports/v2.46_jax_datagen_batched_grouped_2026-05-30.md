# Relatório — v2.46 — Produtização da geração de dataset no caminho batched JAX

> **Data**: 2026-05-30
> **Backlog (SSoT)**: `A-jax-gpu-data-gen` (ROADMAP §0, P1) — fecha a lacuna estrutural
> **Hardware**: RTX A6000 local · Paridade Fortran **<1e-13 c128 PRESERVADA**

---

## Resumo Executivo

Corrige a **lacuna crítica** identificada na análise do simulador JAX GPU: o gerador de
dataset (`SyntheticDataGenerator.generate_batch`) fazia **loop Python por-modelo**, não
usava o caminho batched, gerava **geometria heterogênea** (que cairia no kernel *unified*
~7× lento + OOM) e só suportava **1 freq / 1 TR**. A correção introduz o primitivo
`simulate_multi_jax_batched_grouped` (agrupamento por geometria → bucketed por grupo) e
refatora `generate_batch` para usá-lo, com **multi-config** (freq×dip×TR), **`n_geometries`**
(geometrias batcháveis) e **`jax_chunk_size_models`** (controle de VRAM). Validação:
**22/22** testes do gerador + **53/53** paridade/regressão JAX + revisão multi-agente **SAFE**.

| # | Lacuna | Correção | Arquivo |
|:-:|:-------|:---------|:--------|
| 1 | Loop por-modelo, não-batched | jax → `simulate_multi_jax_batched_grouped` | `data/synthetic_generator.py` |

## 2. Features / API

| Item | Descrição |
|:-----|:----------|
| `simulate_multi_jax_batched_grouped(rho_h, rho_v, esp, positions_z, *, frequencies_hz, tr_spacings_m, dip_degs, cfg)` | Particiona o batch por `esp` idêntico → `simulate_multi_jax_batched` (bucketed) por grupo → reassembla H na ordem original. Retorna `(H_tensor (n_models,nTR,nAng,n_pos,nf,9), info)`. Exportado em `geosteering_ai.simulation`. |
| `generate_batch(..., frequencies_hz=, dip_degs=, tr_spacings_m=, n_geometries=, jax_chunk_size_models=)` | jax → grouped batched; numba → `simulate_multi` por modelo. Shape `(n_models,n_pos,nf,9)` (single-config, **compat retroativa**) ou `(n_models,nTR,nAng,n_pos,nf,9)` (multi-config). `metadata["n_geometry_groups"]` = diagnóstico de batchabilidade. |
| `n_geometries=K` | Amostra K geometrias distintas replicadas (round-robin) → modelos compartilham `esp` → **destrava o bucketed** (1.5–1.9× Numba). `None` = único por modelo (cada um é um grupo). |

### Como usar (cenário de treino 30k — recomendação medida)

```python
cfg = PipelineConfig(simulator_backend="jax")
gen = SyntheticDataGenerator(cfg)
batch = gen.generate_batch(
    n_models=30_000, n_positions=600, n_layers=5,
    frequencies_hz=[20e3, 50e3, 100e3], dip_degs=[0., 30., 60.], tr_spacings_m=[0.5, 1.0],
    n_geometries=256,            # geometrias compartilhadas → bucketed rápido
    jax_chunk_size_models=32,    # limita VRAM (a6000 48 GB)
)
# H_tensor: (30000, 2, 3, 600, 3, 9) ; metadata["n_geometry_groups"] == 256
```

## 3. Correções da Revisão Multi-Agente (6 findings minor)

| Lente | Finding | Correção |
|:------|:--------|:---------|
| Code | `positions_z` usava só `esp_all[0]` | grid agora cobre o modelo **mais espesso** (`esp_all.sum(axis=1).max()`) |
| Adv | sem `jax_chunk_size_models` → OOM em grupo grande (caso 30k) | param plumado p/ o `sim_cfg` jax |
| Adv | teste de paridade `<1e-9` (invariante é `<1e-10`) | apertado p/ **`<1e-10`** (passa) |
| Code | docstring superafirmava "batched" | esclarece jax-batched vs numba-per-model |
| Adv | multi-config `dat_22col` cobre só config 0 | `logger.warning` (H_tensor completo preservado) |
| Physics | `export_per_model` colidiria nomes entre grupos | guard: desabilita no helper agrupado (`dataclasses.replace`) |
