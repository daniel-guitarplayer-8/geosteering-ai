## 1. Árvore de decisão (codificada em `_resolve_backend`)

```
backend="auto":
  ├─ sem GPU JAX                          → Numba 16w×4t
  ├─ n_models < 32 (GPU subocupada)       → Numba 16w×4t
  ├─ geometria não-agrupável (>50% únicos)→ Numba 16w×4t
  └─ GPU + n≥32 + agrupável               → JAX bucketed (grouped)

backend="jax" (forçado): preserva Sprint A (warns + numba_fallback p/ não-agrupável)
backend="numba" (forçado): Numba

GUARD: jax_strategy="unified" + high-config (n_pos≥300, n_configs≥9) → ValueError (OOM)
```

| Ramo da árvore (medido em teste) | Resultado |
|:---------------------------------|:----------|
| auto + GPU + n=64 + homogêneo | → **jax** ✅ |
| auto + GPU + n=16 (< 32) | → numba ✅ |
| auto + GPU + n=64 + heterogêneo | → numba ✅ |
| auto + sem GPU | → numba ✅ |
| jax forçado + heterogêneo + fallback | → numba ✅ |
| jax forçado + heterogêneo + `numba_fallback=False` | → jax ✅ |
| `jax_strategy="unified"` + high-config | → **ValueError** ✅ |
