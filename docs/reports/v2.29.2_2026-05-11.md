## 1. Resumo Executivo

A v2.29.1 corrigiu o bug `NUMBA_NUM_THREADS` no PAI; agora a v2.29.2 endereça
dois problemas adicionais reportados pelo usuário:

1. **Cache LRU hardcoded em 500 MB**: simulações multi-frequência × multi-ângulo

   tentar re-abrir um experimento do histórico, o usuário recebia diálogo

   mas não implementada.
2. **Varredura completa do Simulation Manager**: identificar regressões
   pós-v2.29 e aplicar correções. Encontrada falha de smoke test pré-existente
   (`is_paused`/`is_cancelled` viraram métodos em v2.29 mas o smoke test ainda
   os tratava como properties).

**Conclusão sobre adaptação v2.21 → atual**: investigação confirmou que a
**estrutura GUI v2.21 já está integralmente adaptada** no código atual. As 12
classes principais (`MainWindow`, `SimulationParametersPage`, `BenchmarkPage`,
`ResultsPage`, `PreferencesPage`, `PlotComposerDialog`, etc.) estão preservadas.
A única remoção (warmup background) foi intencional na v2.29 e o bug associado
foi resolvido em v2.29.1. Não há trabalho de adaptação remanescente.

## 2. Causa-Raiz

### 2.1 Cache LRU 500 MB hardcoded

Localizações:
- [`sm_plot_cache.py:73`](../../geosteering_ai/simulation/tests/sm_plot_cache.py#L73) — `max_bytes: float = 500e6` (default do construtor)
- [`simulation_manager.py:7368`](../../geosteering_ai/simulation/tests/simulation_manager.py#L7368) — `LRUPlotCache(maxlen=3, max_bytes=500e6)` (instância)
- [`simulation_manager.py:8603`](../../geosteering_ai/simulation/tests/simulation_manager.py#L8603) — string da mensagem com "(500 MB)" + "(futuro)"

**Cálculo empírico** do tensor multi-freq × multi-angle:

```
shape = (n_models=1000, nTR=2, nAng=4, n_pos=600, nf=4, 9 componentes EM)
dtype = complex128 (16 bytes/elem)
Total = 1000 × 2 × 4 × 600 × 4 × 9 × 16 = 2,764,800,000 bytes ≈ 2.77 GB
       → 5.53× maior que o limite de 500 MB
```

Em produção (8 freqs × 8 dips × 600 pos), apenas ~90 modelos cabem em 500 MB.
Qualquer simulação maior dispara auto-eviction no `put()`, e a re-abertura
pelo histórico cai no diálogo de erro.

### 2.2 Smoke test pré-existente quebrado

`SimulationThread.is_paused()` e `is_cancelled()` viraram métodos em v2.29 (Back
to Basics), mas o smoke test em
[`simulation_manager.py:10515-10527`](../../geosteering_ai/simulation/tests/simulation_manager.py#L10515-L10527)
ainda invocava `st.is_paused is False` (comparação contra a referência do
método, sempre False) — gerando 3 falhas estáticas no smoke test desde v2.29.

### 3.2 Função `default_max_bytes()`

```python
def default_max_bytes() -> float:
    """Estratégia: 10% RAM total, piso 500 MB, teto 4 GB.

    Fallback: 500 MB se psutil ausente ou virtual_memory() falhar."""
    try:
        import psutil
        ram_bytes = float(psutil.virtual_memory().total)
        target = 0.10 * ram_bytes
        return max(500e6, min(target, 4e9))
    except Exception:
        return 500e6  # comportamento histórico v2.4c–v2.29.1
```

Tabela de comportamento:

| RAM Total | 10% RAM | Após piso/teto |
|:---------:|:-------:|:--------------:|
| 4 GB | 400 MB | **500 MB** (piso) |
| 8 GB | 800 MB | 800 MB |
| 16 GB | 1.6 GB | **1.6 GB** |
| 32 GB | 3.2 GB | 3.2 GB |
| 64 GB | 6.4 GB | **4 GB** (teto) |
| 128 GB | 12.8 GB | 4 GB (teto) |

### 3.3 UI Preferências — Group "Cache LRU de Tensores"

Adicionado na `PreferencesPage` entre "Caminhos" e "Estilo de Plot":

- `QSpinBox "Limite total"` (range [100, 16384] MB, default = `default_max_bytes()` em MB)
- `QSpinBox "Snapshots máximos"` (range [1, 20], default 3)
- `QPushButton "Auto-detect"` — reseta limite ao default
- `QLabel` informativo sobre trade-off RAM vs. re-execução

Signal `cache_settings_changed = Signal(int, int)` emitido em cada mudança;
handler `_on_cache_settings_changed`:

1. Valida valores (piso 100 MB, mínimo 1 snapshot)
2. Cria novo `LRUPlotCache` com novos limites
3. **Preserva snapshots existentes** via re-put em ordem original (LRU re-aplica política)
4. Atualiza widgets dependentes: `_update_cache_status`, `_refresh_results_experiment_combo`, `mark_history_out_of_cache`
5. Toast notification confirmando mudança

Persistência em `QSettings("Geosteering AI", "Simulation Manager")`:
- `cache/max_bytes_mb` (int)
- `cache/maxlen` (int)

## 5. Insights / Lições Aprendidas

1. **Hardcoded magic numbers em produção viram dívida técnica**: 500 MB era
   apropriado em v2.4c (Apr/2026 — tensores menores), mas regressão acumulada
   ao adicionar multi-freq + multi-dip não foi acompanhada de revisão do
   limite. Solução: auto-detect baseado em recurso real.

2. **"Feature futura" prometida sem ETA é red flag**: a mensagem "Aumente o
   limite em Preferências (futuro)" pendurava-se desde v2.5 (Abril 2026). 13
   meses depois, a feature surgiu sob a pressão de um bug reportado. Lição:
   features de UX pequenas (~150 LOC) não devem ficar como "futuro".

3. **Smoke test não-versionados acumulam debt**: o teste `is_paused is False`
   silenciosamente falhou desde v2.29 (Maio 2026). Recomenda-se Sprint v2.27
   priorizar cobertura pytest-qt do SM para evitar essa categoria.

4. **CodeRabbit é eficaz para validar fix abrangente**: identificou 4 issues
   ortogonais em um único review (exception coverage, UX consistency, test
   simplification, GUI lifecycle hooks). Worth running antes de commits major.
