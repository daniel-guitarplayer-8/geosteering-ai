## 2. Resultados Empíricos (Intel i9-9980HK, 5 runs × 4 configs × 3 cenários)

| Cenário | v2.22.4 FLAT default | v2.37 heurística | Δ% vs v2.22.4 | +auto-tile | Δ% +tile |
|:--------|--------------------:|-----------------:|--------------:|-----------:|---------:|
| E (single-combo)             | 134.943 mod/h | **145.105 mod/h** | **+7.53%**  | **157.043 mod/h** | **+16.38%** |
| F (4 combos multi-freq)      |  48.541 mod/h |  50.759 mod/h | +4.57%  |  47.513 mod/h | −2.12% |
| B (12 combos multi-array)    |  36.768 mod/h | **43.447 mod/h** | **+18.17%** | **45.784 mod/h** | **+24.52%** |

**Conclusões**:
- Cenário E: **recupera o ganho** identificado em v2.36 D5. Com auto-tile
  ativado, ultrapassa o baseline v2.22.4 em +16% (sem ativá-lo: +7.5%).
- Cenário B: ganho **inesperado** de +18-25%. Combinação FLAT (12 combos
  amortiza overhead) + tile (boa localidade L1 para 200 posições).
- Cenário F: neutro (variação <5%, dentro de variância). Heurística não
  prejudica casos multi-freq.

**Variância**: ~5-10% entre runs (Intel i9 com thermal throttling).
Médias estatisticamente robustas com 5+ runs.

### 3.2 F2 — Auto-detect tile_size

**Arquivo**: `geosteering_ai/simulation/config.py`

Nova função module-level + campos:

```python
def recommend_tile_size(n_positions: int) -> int:
    """Heurística conservadora validada Intel i9 + Apple Silicon."""
    if n_positions <= 0:
        return 2
    if n_positions <= 64:
        return 2
    return 8

@dataclass(frozen=True)
class SimulationConfig:
    tile_size_auto: bool = True  # default ON
    tile_size: int = 4  # ignorado quando tile_size_auto=True
```

**Arquivo**: `geosteering_ai/simulation/multi_forward.py` (linhas 1197-1211)

```python
if cfg.use_tiled_positions:
    if cfg.tile_size_auto:
        from geosteering_ai.simulation.config import recommend_tile_size
        _effective_tile = recommend_tile_size(n_pos)
    else:
        _effective_tile = cfg.tile_size
    _simulate_positions_njit_cached_tiled(..., _effective_tile)
```

**Decisão de design**: tile=8 universal para `n_pos > 64`. Tile=16 mostrou
regressão −5% a −30% em vários perfis (small @ tile=16: −30%; large
@ tile=16: −5% no Intel i9). Heurística conservadora prioriza
robustez sobre ganho marginal.

### 3.3 F3 — Refactor `cli/main.py` → `cli/_main.py`

**Motivação**: `__init__.py` fazia `from geosteering_ai.cli.main import main`,
provocando shadowing do atributo `geosteering_ai.cli.main` entre
módulo (`main.py`) e função (`main()`). Em ordens de import específicas,
`test_cli_main_exports_main` recebia o módulo em vez da função.

**Solução**: `git mv geosteering_ai/cli/main.py geosteering_ai/cli/_main.py`
+ atualização de 6 referências:
- `geosteering_ai/cli/__init__.py:58`
- `geosteering_ai/cli/__main__.py:24`
- `geosteering_ai/cli/warmup.py:208, 218`
- `tests/test_simulation_numba_specializations.py:191`
- `tests/test_cli_mvp.py:30, 73, 140`

Sufixo `_` indica privacidade de submódulo — usuários devem importar
apenas via `from geosteering_ai.cli import main`.
