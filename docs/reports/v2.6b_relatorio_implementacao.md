## 2. Modificação A1 — Checkbox de tema do canvas

### Solicitação do usuário

> "Na seção de Resultados, apresente uma opção (uma checkbox), onde o usuário possa escolher plotar os gráficos em um fundo branco (com os padrões de cores similares aos que eram antes) ou manter o padrão de plotagem atual."

### Implementação

- **Checkbox** `check_canvas_white` adicionado no topo da [`ResultsPage`](../../geosteering_ai/simulation/tests/simulation_manager.py) (logo abaixo do heading "Resultados — Visualização Interativa").
  - Texto: "Fundo branco no canvas (paleta clássica pré-dark)"
  - Tooltip HTML rico explicando as duas modalidades.
- **Slot** `_on_canvas_theme_toggled(checked)` em ResultsPage:
  - Atualiza `self._style.theme = "light"` ou `"dark"`.
  - Chama `apply_style()` + `canvas.set_style()`.
  - Sincroniza com `combo_theme` em PreferencesPage via `blockSignals()`.
  - Re-plota se há simulação corrente.
  - Persiste em `QSettings("GeosteringAI", "SimulationManager")` chave `canvas/theme`.
  - Emite Toast não-bloqueante.
- **Combo `combo_theme`** adicionado em PreferencesPage (Estilo de Plot → "Tema do canvas") com 3 opções (`auto`, `dark`, `light`).
  - Sincroniza bidirecionalmente com o checkbox de A1.
  - Aplica imediatamente sem precisar clicar em "Salvar".
- **Restaurar padrões** (`_on_reset`) atualizado para incluir `combo_theme` + confirmação destrutiva via `QMessageBox.question`.
- **`_collect_style`** agora inclui `theme=self.combo_theme.currentText()` na construção do `PlotStyle`.

### Como funciona

A função `apply_style()` (em [`sm_plots.py`](../../geosteering_ai/simulation/tests/sm_plots.py)) já honra `theme="light"` retornando `bg=#ffffff, fg=#000000, grid=#cccccc` com as cores clássicas (`color_real=#1f4ea8`, `color_imag=#a3272f`) preservadas no `PlotStyle` dataclass. Marcar o checkbox restaura visualmente o canvas pré-v2.6a.

## 3. L4 — Crosshair Sincronizado + EnsembleAnimationBar

### L4-1: CrosshairManager (blitting matplotlib)

- **Wire em ResultsPage**:
  - `_toggle_crosshair()` cria lazy o gerenciador na primeira chamada.

  - Botões `⏮ ◀ ▶ ▶ ⏭`
  - `QSlider(Horizontal)` 0..n_models-1
  - `QDoubleSpinBox` velocidade (0.5–60 fps)

## 4. L5 — CorrelationAnalysisDialog + EnsembleAnalysisDialog

### Estendido [`sm_correlation.py`](../../geosteering_ai/simulation/tests/sm_correlation.py) (+~400 LOC)

#### `CorrelationAnalysisDialog(QDialog)`

- Combo método: `pearson` / `spearman` / `kendall`.
- Checkbox: usar `Re(H)` (default) ou `|H|`.
- Heatmap matplotlib via `imshow` com cmap divergente `RdBu_r`, `vmin=-1, vmax=1`.
- Anotação numérica em cada célula (texto branco/preto conforme magnitude).
- `QTableWidget` com valores formatados (`±.4f`).
- Double-click em célula → exibe valor formatado em label de status.
- Botão "Recomputar" para alternar método sem fechar o dialog.

#### `EnsembleAnalysisDialog(QDialog)`

- Combos de percentil baixo (1, 5, 10, 25) e alto (75, 90, 95, 99).

- Plot vertical (profundidade no eixo Y, invertido):
  - Linha azul: mediana
  - `fill_betweenx` cinza alpha 0.3 entre P5/P95
  - Outliers (z-score > 3) plotados em vermelho com transparência
- Status label informa nº de outliers detectados.

## 5. L6 — Backend Selector + swap_backend runtime

### combo_plot_backend em PreferencesPage

- Lista apenas backends instalados via `available_backends()` de [`sm_plot_backends/`](../../geosteering_ai/simulation/tests/sm_plot_backends/) (lazy import por backend).
- Default `matplotlib` se `QSettings("plot/backend")` não definido.
- Tooltip HTML rico descrevendo cada backend (matplotlib/pyqtgraph/plotly/vispy).
- Slot `_on_backend_changed(backend)` em PreferencesPage:
  - Persiste em `QSettings("plot/backend", value)`.
  - Chama `ResultsPage.swap_backend(backend)`.
  - Atualiza `lbl_status_plot_backend` na status bar.
  - Toast informativo.

### swap_backend(backend) em ResultsPage

- Pré-validação via `make_canvas(backend, parent=None)` (descarta o resultado).
- Para `matplotlib` (default): no-op — canvas atual EMCanvas já é matplotlib.
- Para outros backends: emite warning toast indicando que refator completo das funções `plot_*` para usar `PlotCanvas` API virá em v2.6c (backlog). Por ora, plots permanecem matplotlib mesmo com backend escolhido — mas a infraestrutura de troca está validada.
- Reverte automaticamente para `matplotlib` se backend escolhido falhar.

## 14b. Bug fixes pós-feedback (segunda rodada 2026-04-26)

Após a primeira rodada de validação do usuário com a UI rodando, foram reportados 3 bugs visuais e demanda de visibilidade das features. Todos resolvidos:

### Bug A1 — Layout shift em ParametersPage com Manual / CollapsibleGroupBox

- **Sintoma:** ao trocar para perfil "Manual", os widgets da coluna Parâmetros se deslocavam X/Y. O `CollapsibleGroupBox` envolvendo "Filtros de Hankel" causava reflow.
- **Fix:** revertido o wrap `CollapsibleGroupBox` em torno de `grp_filt`. O grupo voltou a ser adicionado direto a `root.addWidget(grp_filt)` como na versão anterior. PHYSICAL_TOOLTIPS preservados (não causam reflow — apenas `setToolTip`).

### Bug A2 — Geosinais incluindo plot de resistividade

- **Sintoma:** plot "Geosinais (USD/UAD/UHR/UHA)" exibia perfil de resistividade no painel à esquerda mesmo quando o usuário não pediu.
- **Causa raiz:** `_include_rho_in_plot=True` (default v2.5+ para tensor) era reaproveitado no callsite de `plot_geosignals`, então ρ aparecia sempre.
- **Fix:** o callsite passa `include_resistivity = (_layout_key == "geo_nx2_rho")` — ρ só aparece quando o usuário escolheu esse layout via PlotComposerDialog.

### Bug A3 — Canvas não redesenha corretamente ao alternar tema

- **Sintoma 1:** marcar "Fundo branco" deixava parte do canvas branca em uma área pequena, mantendo o resto em dark.
- **Sintoma 2:** desmarcar para voltar ao dark mostrava variação de cores e fonte menor.
- **Causa raiz:** `EMCanvas.set_style()` usava sempre `style.background` (=#ffffff) ignorando `style.theme`. Não atualizava `axes.facecolor` nem disparava `draw_idle()`.
- **Fix:** `set_style()` agora honra `style.theme` (light/dark/auto), atualiza `figure.facecolor`, `ax.facecolor`, spines, ticks, labels, título de **todos** os axes existentes, ajusta a `QPalette` do widget Qt para evitar moldura, e força `draw_idle()`. Também atualiza visualmente axes de plots já renderizados antes do toggle.
