## Resumo Executivo

A Sprint v2.30 implementa **personalização multi-dimensional do CLI Geosteering AI**. Os subcomandos `simulate` e `benchmark` agora aceitam flags para **frequências arbitrárias**, **ângulos de dip variados** e **espaçamentos transmissor-receptor customizados**, eliminando a restrição anterior que forçava uso de valores pré-definidos ou código Python direto.

| Flags `simulate` | 6 | 9 (+3: freq, dips, TR) |
| Flags `benchmark` | 5 | 8 (+3: override freq, dips, TR) |
| Cenários benchmark | 6 (A–F) | 7 (A–G, G=máxima combinatória) |
| Flexibilidade CLI | Baixa (pré-sets) | Alta (combinações arbitrárias) |

---

## Antecedente (Problema Resolvido)

**Situação Anterior (v2.24–v2.29)**:
- CLI `geosteering-cli simulate --models 100` usava defaults hardcoded:
  - Frequência: 20000 Hz (único valor)
  - Dip: 0° (único valor)
  - TR: 1.0 m (único valor)
- Para cenários com múltiplas frequências (2000, 20000, 100000, 400000 Hz), múltiplos dips (0°, 15°, 30°, 45°), ou múltiplos TRs (0.5, 1.0, 1.5, 2.0 m), usuário era forçado a:
  1. Editar código Python direto (chamando `simulate_multi` manualmente)
  2. Executar múltiplas instâncias CLI sequenciais (ineficiente)
  3. Ou aguardar feature no roadmap

**Impacto**: Reduz produtividade — benchmark de Cenário G (4 freq × 4 dips × 4 TR) exigiria 64 chamadas CLI ou edição manual.

### 4. Helper `_parse_float_list()` — Robustez

**Localização**: `geosteering_ai/cli/simulate.py:46-75` e `benchmark.py:72-102`

Implementado **localmente em ambos módulos** (duplicação intencional) para preservar **lazy imports independentes** entre subcomandos.

```python
def _parse_float_list(text: str | None, default: list[float]) -> list[float]:
    """Interpreta string CSV de floats; retorna ``default`` se None ou inválido.

    Aceita vírgula ou ponto-e-vírgula como separador. Espaços são ignorados.
    Trata edge cases: input vazio, apenas separadores, valores não-numéricos.

    Args:
        text: string no formato ``"1.0,2.0,3.0"`` ou ``None``.
        default: lista retornada quando ``text`` é None, vazio ou inválido.

    Returns:
        Lista de floats parseados, ou ``default`` em caso de erro.

    Example:
        >>> _parse_float_list("0, 15, 30", [0.0])
        [0.0, 15.0, 30.0]
        >>> _parse_float_list(",,,", [20000.0])
        [20000.0]  # Edge case: apenas separadores → default
    """
    if not text or not text.strip():
        return default
    try:
        parsed = [
            float(v.strip()) for v in text.replace(";", ",").split(",") if v.strip()
        ]
        return parsed if parsed else default
    except ValueError:
        logger.warning(
            "Valor inválido em lista de floats: %r — usando default %s", text, default
        )
        return default
```

**Melhorias v2.30**:
- ✅ Trata input `None`, vazio ou whitespace → retorna `default`
- ✅ Suporta separadores: `,` e `;` (ambos com espaços opcionais)
- ✅ Edge case ",,," (apenas separadores) → retorna `default` (não `[]`)
- ✅ Logging no ValueError para debugging
