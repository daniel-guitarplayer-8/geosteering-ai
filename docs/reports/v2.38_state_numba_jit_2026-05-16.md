## 4. Dispatcher de Execução — Fluxo

```
simulate_multi(models=[...], cfg)
       │
       ▼
detect_cpu_topology() → (8 cores físicos, 16 lógicos, HT=True)
       │
       ▼
recommend_default_parallelism() → (n_workers=4, threads_per_worker=2)
                                  evita oversubscrição (KB-019)
       │
       ▼
_detect_mode(n_workers, threads_per_worker) → A/B/C/D
       │
       ▼
ProcessPoolExecutor + warmup JIT cooperativo
       │
       ▼
Para cada modelo: simulate_multi_core(cfg, model)
       │
       ├─ cfg.use_flat_prange=True (default) e n_combos ≥ 2
       │    → _simulate_combined_prange_flat (outer prange 4D nTR×nAng×n_pos×nf)
       │
       ├─ cfg.use_flat_prange=True e n_combos < 2 (Cenário E single-combo)
       │    → _simulate_combined_prange (outer prange 2D, sem dispatcher 4D overhead)
       │
       ├─ cfg.use_tiled_positions=True (opt-in path single-TR legado)
       │    → _simulate_positions_njit_cached_tiled
       │      tile_size = recommend_tile_size(n_pos) se tile_size_auto
       │
       └─ caso legacy
            → _simulate_positions_njit_cached
       │
       ▼
H_stack: (n_models, nTR, nAngles, n_pos, nf, 9) complex128
```

### 5.1 Heurísticas adaptativas em vigor

- **FLAT/não-FLAT** (v2.37 F1): `flat_prange_min_combos=2` força não-FLAT em single-combo, recuperando +15% em Cenário E.
- **Tile-size auto** (v2.37 F2): `recommend_tile_size(n_pos)` → tile=2 (n≤64) ou tile=8 (n>64), validado em Intel L1=32kB.
- **Threads HT-aware** (v2.17): `recommend_default_parallelism()` retorna `(n_phys/2, 2)` em CPUs com HT, evitando oversubscrição.
