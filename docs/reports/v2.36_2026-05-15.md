real**, não detectada pelo hook porque o baseline foi medido com FLAT.

## 6. Próximos Passos (Sprint v2.37+ propostos)

### 6.1 Crítico — Recuperar Cenário E (−15% regressão FLAT)

**Descoberta v2.36 D5**: `use_flat_prange=True` (default desde v2.22.4)
regride Cenário E em −15% vs caminho não-FLAT. Cenário E é o caminho
**mais comum em produção LWD**.

**Proposta v2.37**: heurística no dispatcher:

```python
# multi_forward.py
n_combos = nTR * nAngles * len(freqs_hz)
if cfg.use_flat_prange and n_combos == 1:
    # FLAT 4D paga overhead sem ganho — usa não-FLAT para single-combo
    use_flat_effective = False
else:
    use_flat_effective = cfg.use_flat_prange
```

Ganho esperado: **+15% Cenário E** (path produção). Esforço: ~2 horas.

### 6.2 Médio — Auto-detect tile_size ideal

**Estado v2.36**: tile/block validado (paridade bit-exata) mas é opt-in
porque tile_size ótimo depende do perfil:
- small (100 pos): tile=8 → +20%
- medium (200 pos): tile=16 → +11%
- large (600 pos): tile=8 → +47%

**Proposta v2.37**:
- Heurística: `tile_size = max(2, min(64, npt // 32))` (cobre cache L1).
- Validar em 5–10 perfis adicionais antes de promover default `True`.

### 6.3 Baixo — Refatorar `cli/__init__.py`

Pre-existing test-ordering issue `test_cli_main_exports_main`. Renomear
submódulo `cli/main.py` → `cli/_main.py` para evitar shadowing.
