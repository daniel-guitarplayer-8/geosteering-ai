## Resumo Executivo

A v2.10 elimina a lentidão da primeira simulação Numba (3× mais lenta que as
subsequentes) via pool persistente de workers com inicializador JIT embutido.
O `ProcessPoolExecutor` deixa de ser criado por simulação e passa a ser um
singleton reaproveitado entre chamadas; os workers realizam spawn + import +
warmup JIT uma única vez por sessão, via `_numba_init_worker`. A correção
CodeRabbit adiciona fallback conservador correto para p-value combinado em
`sm_correlation.py`.

## 8. Notas de Migração / Breaking Changes

- **`release_numba_pool()`**: nova função pública exportada por `sm_workers`. Deve ser chamada em `closeEvent` de qualquer janela que use o pool persistente. A `MainWindow` já chama automaticamente.
- **`ProcessPoolExecutor` não é mais encerrado** ao término de cada simulação. Workers sobrevivem entre simulações — isso é intencional. O pool é liberado apenas em `closeEvent` ou ao mudar `n_workers`/`n_threads`/`hankel_filter`.
- **Primeira sessão pós-update Numba**: comportamento idêntico a v2.9 — `NumbaPrimer` compila o cache; `_prewarm_numba_pool` dispara workers após conclusão; 2ª simulação é imediata.
