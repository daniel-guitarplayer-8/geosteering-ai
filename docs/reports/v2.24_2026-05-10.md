## 2. Frente 1 — Sprint v2.24 (Débitos Técnicos)

### 2.1 Script PASS count automatizado (1.1)

**Arquivo criado**: `scripts/count_pytest_pass.py` (Python 3.13)

**Origem do débito**: M2 da revisão final v2.23 — contagem PASS era atualizada
manualmente em CLAUDE.md / CHANGELOG.md / ROADMAP.md, levando a divergências
(ex: CLAUDE.md "1604+" vs suite real "1624").

**Implementação**:
- Executa `pytest tests/ -q --tb=no` em subprocess (timeout 30 min)
- Regex extrai contagens de `passed`, `skipped`, `failed`, `xfailed`, `error`
- Output canônico: `"1624 PASS / 295 SKIP / 0 FAIL"` (compatível com docs)
- Modo `--json` para automação CI
- Modo `--from-file` (ler output existente — útil para testes)

**Exemplo de uso**:

```bash
$ python scripts/count_pytest_pass.py
1624 PASS / 295 SKIP / 0 FAIL

$ python scripts/count_pytest_pass.py --json
{"passed": 1624, "skipped": 295, "failed": 0, "xfailed": 0, "errors": 0}
```

### 2.3 `cache=True` explícito em todos `@njit` (1.3)

**Arquivos modificados** (4):

| Arquivo | Funções | Antes | Depois |
|:--------|:--------|:------|:-------|
| `_numba/propagation.py` | `common_arrays`, `common_factors` | `@njit` | `@njit(cache=True)` |
| `_numba/dipoles.py` | `hmd_tiv`, `vmd` | `@njit` | `@njit(cache=True)` |
| `_numba/rotation.py` | `build_rotation_matrix`, `rotate_tensor` | `@njit(fastmath=True)` | `@njit(cache=True, fastmath=True)` |
| `_numba/hankel.py` | `prepare_kr`, `integrate_j0`, `integrate_j1`, `integrate_j0_j1` | `@njit(fastmath=True)` | `@njit(cache=True, fastmath=True)` |
| `_numba/geometry.py` | `find_layers_tr`, `layer_at_depth` | `@njit(fastmath=True)` | `@njit(cache=True, fastmath=True)` |

**Total**: 12 decoradores atualizados. Verificação automatizada via
`test_all_njit_decorators_have_cache_true` (grep recursivo em `_numba/*.py`).

**Benefício**: `cache=True` faz Numba persistir bytecode compilado em
`__pycache__/`, evitando recompilação em re-imports do mesmo processo Python
ou em sessões subsequentes (mesmo hash do AST). Acelera startup em ~3-10s
após a primeira compilação.

**Risco**: Hash do AST kernel determina cache key — não afeta semântica.
Validado pelo gate de paridade Fortran <1e-12 (13/13 PASS).

### 2.4 `use_fastmath` documentado como status documental (1.4)

**Arquivo modificado**: `geosteering_ai/simulation/config.py`

**Decisão conservadora**: dispatcher real do `cfg.use_fastmath` (compilar 2
versões `_fast`/`_safe` e selecionar via wrapper Python) foi **deferido para
v2.25+**. Sprint v2.24 apenas refina a docstring esclarecendo:

1. Status atual = puramente documental (decoradores são INCONDICIONAIS)
2. Mudar `use_fastmath` em runtime NÃO altera comportamento
3. Roadmap: dispatcher real seguirá padrão de `cfg.use_flat_prange` em
   `multi_forward.py`
4. Funções NUNCA elegíveis a fastmath: lista explícita das funções raiz e
   da cascata crítica

**Justificativa**: complexidade adicional foi diferida até haver demanda
real por desligar fastmath em runtime (debugging avançado de paridade).
Reduz superfície de risco sem perder o débito (fica documentado e justificado).

## 3. Frente 2 — I2.5 (Hooks de Qualidade)

### 3.1 `check-ptbr-accentuation.sh`

**Arquivo criado**: `.claude/hooks/check-ptbr-accentuation.sh`

**Evento**: PostToolUse Edit|Write (registrado em `.claude/settings.json`)

**Catálogo**: `.claude/ptbr-words.txt` (TSV 2-col, 60+ pares):

```
configuracao	configuração
nao	não
execucao	execução
funcao	função
implementacao	implementação
...
```

**Lógica**:
1. Lê `tool_input.file_path` e `tool_input.new_string` do JSON do hook
2. Filtra arquivos relevantes: `.md`, `.py`, `.sh`, `.f08`, `.f90`
3. Pula paths em whitelist: `legacy/`, `old_geosteering_ai/`, `.backups/`,
   `node_modules/`, `__pycache__/`
4. Para cada par no catálogo, busca palavra sem acento via `grep -wiE`
   (word-boundary, case-insensitive)
5. Emite WARN no stderr para cada match (não bloqueia — exit 0)

**Bypass**: `CLAUDE_BYPASS_PTBR=1` desabilita verificação.

**Exemplo de output**:

```
[ptbr-accentuation] Palavras sem acento detectadas em: /tmp/foo.md
  ⚠ 'configuracao' → deveria ser 'configuração'
  ⚠ 'acentuacao' → deveria ser 'acentuação'
  Regra inviolável CLAUDE.md: documentos PT-BR DEVEM ter acentuação.
```

### 3.2 `generate-pr-description.sh`

**Arquivos criados**:
- `.claude/hooks/generate-pr-description.sh`
- `.claude/templates/pr_description_template.md`

**Disparo**: manual via `bash .claude/hooks/generate-pr-description.sh [base]`
(default `base=main`).

**Lógica**:
1. Verifica que está em git repo
2. Coleta: branch atual, contagem de commits, contagem de arquivos modificados
3. Lista commits formatados (`- <hash> <subject>`)
4. Substitui tokens no template via `awk` (mais robusto que sed para conteúdo
   com `/` e `&`):
   - `{SUMMARY}` → primeiro commit subject
   - `{CHANGES}` → `git diff --name-status` formatado
   - `{BRANCH}`, `{BASE}`, `{COMMITS_LIST}`, `{COMMIT_COUNT}`, `{FILE_COUNT}`
5. Emite Markdown estruturado em stdout (pronto para `gh pr create --body`)

**Template inclui**: Resumo / Mudanças / Test Plan (com checklist de
paridade Fortran) / Referências.

## 4. Frente 3 — I2.6 (CLI MVP)

### 4.1 Estrutura

**Diretório criado**: `geosteering_ai/cli/`

| Arquivo | Propósito |
|:--------|:----------|
| `__init__.py` | Re-export de `main` |
| `__main__.py` | Entry point para `python -m geosteering_ai.cli` |

| `simulate.py` | Handler do subcomando `simulate` |
| `benchmark.py` | Handler do subcomando `benchmark` |

### 4.2 Entry point

**Modificado**: `pyproject.toml`

```toml
[project.scripts]
geosteering-cli = "geosteering_ai.cli:main"
```

Após `pip install -e .`, o comando `geosteering-cli` fica disponível no PATH.

### 4.3 Subcomandos

#### `version`

```bash
$ python -m geosteering_ai.cli version
Geosteering AI Simulation Manager v2.24
```

#### `simulate`

Gera N modelos sintéticos (5-camada, ρ_h ∈ [1, 100] Ω·m, ρ_v/ρ_h ∈ [1, 3])
e executa `simulate_multi` em batch.

```bash
$ python -m geosteering_ai.cli simulate \
    --models 100 --n-pos 600 --workers 4 --threads 2 --seed 42 --out /tmp/sim
```

Opções: `--models N`, `--n-pos`, `--workers`, `--threads`, `--seed`,
`--out DIR`, `--quiet`.

#### `benchmark`

Executa cenário canônico (A/B/C/D/E/F) com warmup + run cronometrado.

```bash
$ python -m geosteering_ai.cli benchmark --scenario A --n 100
Cenário A — 1,180,000 mod/h
```

Cenários:

| Cenário | Característica           | n_pos | nf | nTR |
|:-------:|:-------------------------|:-----:|:--:|:---:|
| A       | Single-pos, 1 freq, 1 TR | 1     | 1  | 1   |
| B       | Multi-pos, 1 freq        | 100   | 1  | 1   |
| C       | Multi-pos, multi-freq    | 100   | 4  | 1   |
| D       | Single-pos, multi-TR     | 1     | 1  | 4   |
| E       | Inv0Dip 0° (default)     | 600   | 1  | 1   |
| F       | Multi-TR + multi-freq    | 100   | 4  | 4   |

### 4.4 Reutilização (DRY)

Funções existentes reutilizadas (sem duplicação):

- `geosteering_ai.simulation.simulate_multi` — API principal de simulação
- `geosteering_ai.simulation.config.SimulationConfig` — auto-detect Sprint v2.23 A.2
- `geosteering_ai.simulation._workers.recommend_default_parallelism` — defaults

### 4.5 Lazy imports

Imports pesados (`numba`, `simulation`) são feitos **dentro dos handlers**
para que `geosteering-cli --help` seja rápido (<5s mesmo no primeiro uso).
Validado por `test_cli_does_not_import_simulation_at_help_time`.

## 8. Decisões e Riscos

### 8.1 Decisões

1. **`use_fastmath` dispatcher real deferido** (v2.25+): docstring esclarece
   status documental; reduz risco e mantém débito documentado.
2. **Hooks PT-BR como WARN não BLOCK**: PT-BR é débito documental,
   não-bloqueante para continuidade do desenvolvimento.
3. **CLI MVP com argparse** (não Click/Typer): alinhado ao padrão dos
   benchmarks v212/v214 do projeto.
4. **Lazy imports**: `--help` < 5s mesmo no primeiro uso.

### 8.2 Riscos mitigados

| Risco | Mitigação aplicada |
|:------|:-------------------|
| `cache=True` quebra paridade Fortran | Hash do AST determina cache key — sem efeito semântico. Validado: 10/10 PASS |
| Hook PT-BR gera ruído em código legado | Whitelist `legacy/`, `old_geosteering_ai/`, `.backups/` |
| CLI overhead em entry point | Lazy imports nos handlers; `--help` < 5s |
| Hook PT-BR pode falhar em ambientes sem `jq` | Fallback `exit 0` em qualquer falha |
