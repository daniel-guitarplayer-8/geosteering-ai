## 6. Bug Crítico Resolvido — D5 Mixed Precision Ordem

### Problema (pré-v2.40)

```python
# geosteering_ai/training/loop.py:914 (antes)
def run(self, model, ...):
    self._setup_mixed_precision()  # ← policy setada DEPOIS
    self.compile(model, ...)
    self.fit(model, ...)

# Resultado: model foi construído com policy fp32, depois set_global_policy
# mudou para mixed_float16 mas as camadas JÁ EXISTEM em fp32.
# Camadas mantém compute_dtype='float32' → SEM GANHO de mp16.
```

### Fix (v2.40)

```python
# Novo helper em geosteering_ai/models/registry.py:
def build_model_with_mp_policy(config):
    from geosteering_ai.training.loop import setup_mixed_precision_policy
    setup_mixed_precision_policy(config)  # ← PRIMEIRO
    model = build_model(config)            # ← DEPOIS (camadas fp16)
    return model

# Uso correto agora:
model = build_model_with_mp_policy(config)
assert model.compute_dtype == "float16"  # ✓
```

### Detecção de uso errado

```python
# geosteering_ai/training/loop.py:run() — warning ativo
if self.config.use_mixed_precision:
    if str(model.compute_dtype) != "float16":
        logger.warning(
            "use_mixed_precision=True mas model.compute_dtype=%s. "
            "Fix: use build_model_with_mp_policy(config) ANTES de run().",
            model.compute_dtype,
        )
```
