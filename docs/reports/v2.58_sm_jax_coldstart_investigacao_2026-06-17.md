## 2. Investigação empírica (RTX A6000, env `Geosteering_AI`, jax 0.7.2)

Caminho FIEL ao SM (`sim_request._run_simulation`) com a config EXATA do usuário:

| Experimento | Wall | Observação |
|:------------|-----:|:-----------|
| Processo 1 — **cold** (shape inédita no cache) | **173 s** | reproduz o ~241 s reportado |
| Processo 2 — **fresh process, mesma config** (`rng_seed=42`) | **29.1 s** | cache de disco HIT total (cache_files inalterado) |
| 2ª chamada **in-process** (probe n=64) | 11.8 s | compute puro é trivial |
| Cold com **K=1** (`n_geometries=1`) | **108 s** | 1 grupo de 1000 → 1 compilação (vs 4) |

Fatos verificados:

```
jax.devices() = [CudaDevice(id=0)]   → JAX roda na GPU, NÃO em CPU
platform = gpu                       → sem fallback p/ CPU
n_geometry_groups = 4                → templates ativos (K=4 → grupos de 250)
H6.shape = (1000, 1, 1, 600, 1, 9)   → saída correta, 1000 modelos
cache = ~/.cache/geosteering/jax_compilation_cache (759 MB, 9227→9320 arquivos)
```

## 3. Causa-raiz (verificada por Workflow adversarial — 4 auditorias + síntese)

| # | Achado | Evidência |
|:-:|:-------|:----------|
| 1 | **Cold-start XLA, não regressão.** 173 s = ~145 s de compilação (~4 programas) + ~28 s. Warm = 29 s (< Numba). | `dispatch.py` resolve `jax`; `_jax/__init__::_setup_xla_environment` cache OK |
| 2 | **Assimetria de warmup.** O SM/CI pré-aquecem o Numba (`geosteering-warmup`, `_warmup_numba_tier2_sync`, init de pool). **Nada** pré-aquece o JAX (só menções em doc-comment no SM/gui). | `warmup_jax_simulator` existe mas só é chamado pelo CLI (`_exec.py`) |
| 3 | **Chave do cache embute o batch-dim por grupo.** SM `K=4`→grupo 250; CLI `max(1,min(n//256,4))`→grupo 333. 250≠333 → o SM não reusa o cache aquecido pelo CLI. | `multi_forward.py:1131-1139` (vmap por grupo); `_exec.py:157-162` |
| 4 | **Warmup genérico é parcialmente eficaz.** `warmup_jax_simulator` usa geometria HOMOGÊNEA → só 1 dos K programas (templates têm `ct/cr` distintos). | `warmup.py:198-218` (rho/esp uniformes) |

> **Conclusão:** a disparidade é o custo ÚNICO de compilação XLA para uma forma de
> batch que o cache nunca tinha visto. Execuções repetidas (mesma forma) já batem o
> Numba (~29 s < 36.8 s) via cache de disco persistente.

## 4. Correções aplicadas (baixo risco, física intocada)

| Arquivo | Mudança |
|:--------|:--------|
| `apps/.../simulation/viewmodel.py` | `run()` emite nota de cold-start honesta p/ backend `jax`/`auto` |
| `apps/.../simulation/view.py` | Expõe o knob **Geometrias (K)** (`SimRequest.n_geometries`, antes sem UI); `0`=auto. Tooltips de cold-start/cache |
| `geosteering_ai/gui/services/sim_request.py` | Docstring `_run_simulation`: documenta o cold-start XLA × Numba (texto) |
| `tests/test_sm_jax_coldstart.py` | 8 testes (mensagem, knob K, round-trip de sessão, wiring da View) |

**Knob de Geometrias (K):** controla o trade-off diversidade × cold-start.
`K=1` → 1 compilação (cold ~108 s); `K=4` (auto) → cold ~173 s; **warm ~29 s p/
qualquer K**. ρₕ/ρᵥ/λ continuam por-modelo (diversidade petrofísica preservada).

> **Decisão deliberada:** NÃO adotamos a fórmula-K crua do CLI (`max(1,min(g//256,4))`)
> como default — ela colapsaria K=1 p/ qualquer `g ≤ 511`, matando a diversidade
> geológica de ensembles médios (regressão científica). O usuário escolhe K via UI.
